# SVG, Satori, and filter gotchas (hard-won — read before building brand art)

Each of these caused a real, silent visual failure during the 2026-07 rebrand.

## Satori / next-og (`ImageResponse`) — OG images, icons

Satori is not a browser. Inside any OG route or `app/icon.tsx`-style file:

- **Flat literal colors only.** No CSS variables, no classes, no `<defs>`, no
  `<linearGradient>`, no `<filter>`. Import hexes from `BRAND`
  (`lib/brand/constants`). This is why the flat mark variant exists.
- **Every multi-child `<div>` needs explicit `display: "flex"`** or Satori
  throws/mis-lays-out.
- Gradient-text headlines work only as `backgroundImage` +
  `backgroundClip: "text"` on spans inside an explicit column-flex `<h1>`.
- Edge runtime: no node built-ins; `lib/brand/*` is safe (dependency-free).
- Use `AstralRose` / `BrandMarkTile` from `lib/brand/glyph` — never inline the
  mark by hand.

## Browser/librsvg SVG (React components, static exports)

- **Gradients on stroke-only geometry:** the default `objectBoundingBox`
  gradient units produce a zero-area box on straight lines (the mark's tick
  strokes) — the stroke silently vanishes. Always
  `gradientUnits="userSpaceOnUse"` with `x1="5" y1="5" x2="27" y2="27"` on the
  shared 0–32 grid.
- **`<use href="#symbol">` defaults to 100% × 100% of the viewport**, not the
  symbol's viewBox size. In any container whose viewBox isn't exactly
  `0 0 32 32`, give the use explicit `width="32" height="32"` or the mark
  silently rescales/offsets (this broke every bordered treatment once).
- **Lighting filters wash out flat faces.** `feSpecularLighting` under a
  distant light adds a uniform sheen across any large flat shape — a coin or
  tile run through the bevel filter turns into a gold smear. Rule: the bevel
  (`pg-pop`) goes on the *glyph group only*; containers get a shadow-only
  filter.
- Relief effects die below ~32px render size; use flat/compact variants there.
- sharp rasterization: density must scale off the SVG's *intrinsic viewBox
  width*, not the target pixel width, or big canvases blow the input pixel
  limit (see `png()` in `scripts/generate-brand-assets.mjs`).

## Filter recipes (canonical copies)

The canonical filter/gradient strings live in `lib/brand/svg.ts`
(`GOLD_GRAD`, `POP_FILTER`, `CARVE_FILTER`, `FLOAT_FILTER`,
`FLOAT_SHADOW_FILTER`) and as React JSX in `components/layout/logo.tsx`.
Copy from there — don't re-derive the numbers. Summary of intent:

| id | What | Where it's allowed |
| --- | --- | --- |
| `pg-gold` | userSpaceOnUse 135° gold gradient | any browser/librsvg SVG |
| `pg-carve` | inner top shadow — "socket chiseled into the screen" | socket circle only |
| `pg-pop` | bevel light az 235°/el 36° + lift shadow | the rose glyph group only |
| `pg-float` | bevel + double drop shadow (Levitant) | glyph directly, no container |
| `pg-float-s` | shadow-only double lift | coins/tiles/containers |

## Card renderer (bake side)

`lib/render/card-image.tsx` bakes brand geometry into stored card PNGs and
the `/api/cards/[id]/og` images. Preview (`components/cards/*`) and bake must
stay pixel-identical. Any change to brand geometry there requires bumping
`CARD_LAYOUT_VERSION` in `lib/cards/layout-version.ts` and a rebake
(`scripts/rebake-renders.mjs`).

## Fonts

- Cinzel via `next/font/google` in app chrome. For asset generation, the
  legacy CSS API (`https://fonts.googleapis.com/css?family=Cinzel:600` fetched
  with a curl-like UA) serves static-instance TTFs that opentype.js can
  outline — the modern API only serves variable/woff2.
- Never ship art that depends on Cinzel being installed: use the outlined
  wordmark path (`lib/brand/wordmark.ts`, regenerated by
  `scripts/build-wordmark-path.mjs`).
